为什么要做这个项目
主要是学习用的，主要是了解一下网盘项目的底层实现原理。注：这个项目是复刻版，原作者是b站up“职坐标”。

这个项目的功能主要有哪些
主要是有两大模块，一个是好友模块另一个是文件模块。前者提供加好友、私聊、群聊以及显示在线好友等等功能。后者提供目录的创建、文件的上传、文件的分享等等功能。具体的显示效果参考“介绍”目录

这个项目是怎么被设计出来的
主要使用C语言设计，框架使用的是qt。技术亮点：
1、使用可变长度结构体来节省资源，协议的其他部分（比如消息的类型、消息的总长度等）是固定的，消息的长度是可变的。
2、配置文件单独放在qt的资源文件里，比较容易维护。
3、缓存技术的应用。发送文件的时候设置了一个特定的缓存容量大小，可以提升发送效率。
4、使用一个QTcpSocket类，这个类面向的是可靠传输。
5、单例模式的使用，一个客户端对应着一个实例，这个单例模式可以实现变量的共享。
6、组合模式的使用，文件模块以及好友模块当成一个组建放入到opewidget（主面板）中，这个好友模块在文件上传的时候也会被用到。
7、工厂模式的使用，创建pdu是通过封装好的方法mkPDU来创建的，算是一种工厂模式。

不足的地方：
1、异常处理不是很到位，比如数据库连接失败的时候没有给出异常提示。
2、跨平台问题没有处理，文件路径的格式都是依据windows进行开发的。

如果要使用这个项目要注意什么
1、文件路径的格式并没有适配linux,所以linux上运行会出现一点问题。
2、使用qt5开发的。
